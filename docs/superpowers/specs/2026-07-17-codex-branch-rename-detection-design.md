# Codex Branch Rename Detection Design

## Problem

The branch-rename interceptor subscribes only to the Claude-specific `Bash` tool name and extracts commands only from `input.command`. Codex ACP command events instead expose the command text as `ToolEvent.toolName`. The interceptor registry uses exact matching for named subscriptions, so it never dispatches a Codex event such as `git branch -m fix-auth-bug`. The workspace therefore retains `isAutoGeneratedBranch: true`, allowing the later pre-push or pre-PR interceptor to replace the agent's chosen branch name.

## Considered Approaches

1. Subscribe to all tool events and use the existing `extractMatchingCommand` helper. This is the recommended approach because it matches the established pre-push/pre-PR pattern and supports commands carried in `command`, `cmd`, `title`, or `toolName` while preserving content-based filtering.
2. Teach the interceptor registry to interpret command-shaped tool names as `Bash`. This broadens registry semantics for every interceptor and couples dispatch to shell parsing unnecessarily.
3. Add dynamically generated command strings to the interceptor's `tools` list. Exact subscriptions cannot enumerate arbitrary commands, so this does not solve the problem.

## Design

Set `branchRenameInterceptor.tools` to `'*'`. Inside `onToolComplete`, call `extractMatchingCommand(event, /\bgit\s+branch\s+-[mM]\b/, logger)` and return when it finds no matching command. Keep the existing error-event guard, Git branch lookup, empty-result handling, and atomic workspace metadata update unchanged.

No registry, ACP translation, workspace service, database, or UI changes are required.

## Error Handling and Edge Cases

- Failed tool executions remain ignored even when their title contains a rename command.
- Claude-style `input.command` events continue to detect `git branch -m` and `git branch -M`.
- Codex-style raw command tool names detect both lowercase `-m` and uppercase `-M`.
- Unrelated wildcard-delivered tool events return without Git or workspace writes.
- Failed or empty `git rev-parse` results continue to skip workspace updates.

## Testing

Remove the command-extraction mock so the tests exercise real event payloads. Add regression assertions that the interceptor uses the wildcard subscription and that a Codex event whose raw `toolName` is `git branch -m codex-name` updates the workspace after resolving the current branch. Retain the existing Claude-style success and negative-path tests with commands supplied in `input.command`. Run the focused test through red and green, then run typecheck, formatting, the full Vitest suite, and the production build.
